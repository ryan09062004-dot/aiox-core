## Story
**As a** usuário do ShapeAI,
**I want** receber um plano alimentar personalizado com 5 refeições diárias gerado por IA com base no meu objetivo, altura, peso e sexo,
**so that** eu tenha um guia nutricional concreto que complementa meu plano de treino e acelera meus resultados.

## Acceptance Criteria

1. **Geração de plano alimentar no ai-engine** — Novo módulo `meal_plan_generator.py` gera plano com 5 refeições (café da manhã, lanche da manhã, almoço, lanche da tarde, jantar) usando a Anthropic API (`claude-sonnet-4-6`). O plano é personalizado por: `goal` (hypertrophy/fat_loss/conditioning/maintenance), `height_cm`, `weight_kg`, `sex` (M/F).

2. **Schema da refeição** — Cada refeição contém: `meal_type` (string), `name` (string), `description` (string), `calories_approx` (int), `protein_g` (int), `carbs_g` (int), `fats_g` (int), `ingredients` (string[]).

3. **Migration de banco** — Nova migration `009_meal_plans.sql` cria tabela `meal_plans` com: `id UUID PK`, `user_id UUID FK users`, `goal VARCHAR`, `height_cm SMALLINT`, `weight_kg SMALLINT`, `sex CHAR(1)`, `meals JSONB`, `generated_at TIMESTAMPTZ DEFAULT NOW()`.

4. **Endpoint de geração** — `POST /meal-plans/generate` no API Gateway: requer autenticação, busca perfil do usuário, chama ai-engine, persiste resultado. Retorna o plano gerado.

5. **Endpoint de leitura** — `GET /meal-plans/latest` retorna o plano mais recente do usuário (sem gerar novo). Retorna 404 se nenhum plano existe.

6. **Freemium** — Usuários Free: apenas 1 geração gratuita. Após isso, responde 402 com `{ error: "SUBSCRIPTION_REQUIRED" }`. Usuários Pro: gerações ilimitadas. Lógica reutiliza `checkFreemiumLimit` adaptada para `meal_plans`.

7. **Tela de plano alimentar** — Nova tela `apps/mobile/app/(app)/meal-plan.tsx` exibe as 5 refeições em cards. Cada card mostra: nome da refeição, nome do prato, descrição, macros (kcal, proteína, carb, gordura) e ingredientes.

8. **Acesso à tela** — Botão/ícone "Plano Alimentar" adicionado à tela home (`index.tsx`) como ação rápida, com ícone de garfo ou prato.

9. **Estado de loading e erro** — Tela mostra `ActivityIndicator` durante fetch. Se não há plano, exibe CTA "Gerar meu plano alimentar" que chama `POST /meal-plans/generate`. Se 402, mostra paywall.

10. **Tipos compartilhados** — Novos tipos `MealItem` e `MealPlan` em `packages/shared/src/types/meal-plan.types.ts`, exportados via `packages/shared/src/index.ts`.

### Padrão de rota no API Gateway

```typescript
// services/api-gateway/src/routes/analyses.ts — padrão
import { pool } from '../db/client'
import { requireAuth } from '../middleware/auth'

export async function mealPlansRoutes(app: FastifyInstance) {
  app.get('/meal-plans/latest', { preHandler: requireAuth }, async (request, reply) => {
    const userId = request.authUser.id
    const { rows } = await pool.query(
      `SELECT * FROM meal_plans WHERE user_id = $1 ORDER BY generated_at DESC LIMIT 1`,
      [userId]
    )
    if (!rows[0]) return reply.status(404).send({ error: 'NOT_FOUND' })
    return reply.send(rows[0])
  })
}
```
